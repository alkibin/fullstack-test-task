## Тестовое задание на позицию Fullstack разработчика (Python + React)

**Вводные:**
1. Здесь представлен MVP проект файлообменника. Он позволяет загружать файлы, проверяет их на подозрительный контент и отправляет алерты;
2. Репозиторий содержит в себе бэкенд и фронтенд части;
3. В обоих частях присутствуют баги, неоптимизированный код, неудачные архитектурные решения.

**Задачи:**
1. Проведите рефакторинг бэкенда, не ломая бизнес-логики: предложите свое видение архитектуры и реализуйте его;
2. (Дополнительно) На бэкенде есть возможность неочевидной оптимизации - выполните ее;
3. (Дополнительно) Разбейте логику фронтенда на слои;

**Запуск:**
1. ```docker compose -f docker-compose.dev.yml up```
2. ```docker exec -it backend alembic upgrade head```


**Открыть фронт:** ```http://localhost:3000/test``` 

**Открыть бэк:** ```http://localhost:8000/docs```



Что исправлено в рамках рефакторинга

- Доменные исключения вместо HTTPException в сервисе — src/exceptions.py
register_exception_handlers(), единая точка маппинга в HTTP-статусы.
- Блокирующий I/O диска обёрнут в asyncio.to_thread — везде, где
сервис/таски трогают диск чтобы не блокировать event loop.
- Celery-инстанс вынесен в отдельный src/celery_app.py — вместо
создания приложения внутри tasks.py вперемешку с бизнес-логикой.
- Все идентификаторы с настройками приложения перенесёны в settings вместо прямого os.environ.get -
единый источник конфигурации для всего приложения.
- Оркестрация селери задач вынесена в handlers.py
- В processing.py вынесена бизнесл логика для воркеров
- Cascade между StoredFile и Alert
- FileUpdate.title — добавлены min_length=1, max_length=255 т.к. в базе ограничение длины строки 255
- Убраны избыточные nullable=False / Integer в моделях —
SQLAlchemy 2.0 выводит их сам из Mapped
- Сервисы и эндпоинты поделены на 2 класса по работе с файлами и алертами.
- Добавлено фастапишное управление зависимостями Depends. 
- Пул соединений с бд открывается при 1 запросе и висит в памяти. Сессия передается в сервис.
- Добавлен main.py как единая точка входа в приложение фастапи
- другие изменения
